# 09. Review Checklist

## PR/변경 검토 공통 체크리스트

### 요구사항

- [ ] 사용자 요청을 정확히 해결했는가?
- [ ] 범위를 벗어난 수정이 없는가?
- [ ] 불필요한 리팩터링이 없는가?
- [ ] 남은 이슈가 명확히 적혀 있는가?

### 코드 품질

- [ ] 기존 스타일과 일관적인가?
- [ ] 중복이 과도하게 늘지 않았는가?
- [ ] 이름이 명확한가?
- [ ] 타입 안정성이 유지되는가?
- [ ] 에러 처리가 있는가?
- [ ] loading/empty/error state가 필요한 경우 반영되었는가?

### 테스트

- [ ] 관련 테스트가 실행되었는가?
- [ ] 실행하지 못한 테스트의 이유가 있는가?
- [ ] 수동 검증 절차가 있는가?
- [ ] 실패 로그가 요약되었는가?

### 보안

- [ ] secret이 코드/로그/client bundle에 포함되지 않았는가?
- [ ] 인증/인가 변경이 검토되었는가?
- [ ] 외부 입력 검증이 있는가?
- [ ] SSRF 가능성이 있는 fetch가 제한되는가?
- [ ] SQL/template/command injection 가능성이 없는가?
- [ ] CORS/CSRF/session/cookie 설정 변경이 안전한가?
- [ ] Docker/infra 권한이 과도하지 않은가?

### 운영

- [ ] rollback 가능성이 있는가?
- [ ] migration이 있다면 적용 순서가 명확한가?
- [ ] config/env 변경이 문서화되었는가?
- [ ] 배포 전 필요한 수동 작업이 적혀 있는가?
